# Security Logging & Alerting Failures
로깅과 모니터링이 없으면 공격과 침해를 탐지할 수 없고 알림이 없으면 신속한 대응이 불가능합니다

### 핵심 문제

- 로그인, 실패한 로그인, 고가치 트랜젝션 등 감사 이벤트가 누락되거나 불일치하게 기록됨
- 경고 에러 발생 시 로그 메시지가 없거나 불명확함
- 로그 무결성이 보호되지 않아 변조 가능
- 애플리케이션 API 로그에 대한 의심 활동 모니터링 미흡
- 로컬에만 로그 저장, 백업 없음
- 침투 테스트가 알림을 트리거 하지 않음
- 실시간 공격 탐지 알림 불가
- 로그에 민감 정보가 포함되거나 공격자에게 노출 됨
- 너무 많은 오탐으로 중요 알림을 구별하지 못함

### 예방법

- 로그인, 접근제어, 서버 입력 검증 실패를 충분한 사용자 컨텍스트와 함께 기록하고 보관
- 보안 컨트롤이 있는 모든 영역에 성공, 실패 여부 모두 로깅
- 로그 관리 솔류션이 쉽게 소비할 수 있는 표준 포맷 사용
- 로그 데이터를 올바르게 인코딩해 로그 인젝션 방지
- 번조, 삭제를 위한 감사 추적 및 무결성 제어 적용
- 의심스러운 활동 시 알림 방송
- 개발자 가이드 수립
- 모니터링 알림 유스케이스 및 플레이북 마련
- DB나 데이터에 허니토큰을 심어 공격자 탐지
- AI 행동 분석 기술을 활용해 오탐률 낮추기

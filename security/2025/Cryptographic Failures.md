# Cryptographic Failures
암호화 실패는 암호화 부재, 취약한 암호화, 암호화 키 유출 등의 문제를 다룹니다 취약한 난수 생성기 관련 문제들이 있습니다

### 주요 취약점

- 오래되거나 취약한 암호화 알고리즘 사용
- 기본 암호화 키 사용이나 약한 키 생성, 키 재사용, 키 관리/로테이션 부재
- 소스 코드 저장소에 암호화 키가 포함된 경우
- HTTPS 미적용 또는 HTTP 보안 헤더 누락
- 서버 인증서 및 신뢰 체인 미검증

### 예방법

- **데이터 분류**: 처리, 저장, 전송 되는 데이터를 분류하고 법에 따라 민감 데이터 식별
- **암호화 적용**: 민감한 데이터는 저장하지 않고 암호화 전송 시 TLS 1.2 이상만 허용 암호 스위트 사용, HSTS 적용, CBC 모드 지원 중단 양자 내성 알고리즘 지원 준비
- **키 관리**: 가장 민감한 키는 HSM에 저장하고 검증된 암호화 라이브러리 사용하기
- **비밀번호 저장:** 강력한 해시 함수 사용 + 솔트 사용

# Insecure Design
안전하지 않은 설계는 누락되거나 비효과적인 통제 설계로 표현되는 다양한 취약점을 포괄하는 넓은 범주입니다
안전하지 않은 설계와 안전하지 않은 구현은 다름
안안전한 설계라도 구현 결함으로 취약점이 생길 수 있고 반대로 안전하지 않은 설계는 완벽한 구현으로 고칠 수 없음

### 핵심 요소

- **요구사항 및 자원관리**: 보호 요구사항을 받아 전체 예산과 기술 요구사항을 계획합니다
- **보안 설계**: 보안 설계는 지속적으로 위협을 평가하고 공격을 방어하도록 생긴 문화이자 방법론
- **보안 개발 생명주기(SDL)**: 자기 책임 문화 육성 및 보안을 모든 중요한 설계 결정에 포함 시키는 마인드셋

### 예방법

- 보안 전문가와 개발 생명주기를 수립 운용하기
- 보안 설계 패턴 또는 검증된 컴포넌트 라이브러리 사용
- 인증, 접근 제어, 비즈니스 로직 등 핵식 영역에 위협 모델링 적용
- 사용자 스토리에 보안 언어와 통제를 통합
- 프론트엔드부터 백엔드까지 각 계층에서 타당성 검사 적용
- 핵심 흐름이 위협 모델에 저항하는지 단위, 통합테스트로 검증
- 노출 및 보호 필요성에 따라 시스템, 네트워크 계층을 계층별로 분리

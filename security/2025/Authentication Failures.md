# Authentication Failures
하드코딩된 비밀번호, 부적절한 인증, 세션 고정 등이 있습니다

### 취약점이 존재하는 경우

- 유출된 계정 목록을 이용한 자동화 공격
- 브루트포스 허용
- 기본 비밀번호 사용
- 이미 유출된 비밀번호로 계정 생성 허용
- 취약한 비밀번호 복구 프로세스
- 약한 해시로 비번 저장
- 다중 인증이 없거나 미흡
- 토큰 미삭제
- 세션 ID 재사용

### 방지법

- MFA 적용
- 패스워드 매니저 사용
- 기본 자격증명 배포 금지
- 취약 비밀번호 차단
- 유출된 비번 목록과 대조
- **NIST 800-63b 가이드라인** 준수
- 의심스러운 상황 아니면 주기적으로 비번 교체 강요 금지
- 계정 열거 공격 방지
- 로그인 실패 횟수 제한 및 로깅
- 검증된 인증 시스템 활용

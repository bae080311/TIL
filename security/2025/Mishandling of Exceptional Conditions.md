# Mishandling of Exceptional Conditions
잘못된 에러처리 논리적 오류 실패 시 허용 등 비정상적인 상황에서 발생하는 문제들을 다룹니다
예외 조건 오처리는 소프트웨어가 비정상적이고 예측 불가능한 상황을 예방 탐지 대응하지 못할 때 발생하며 크래시 예상치 못한 동작 그리고 보안 취약점으로 이어집니다

### 주요 원인

- 입력값 검증 미흡 또는 불완전 검증
- 오류가 발생한 함수가 아닌 상위레벨에서의 늦은 에러처리
- 메모리, 권한,네트워크 이슈 같은 예상치 못한 환경 상태
- 일관성 없는 예외 처리, 또는 아예 처리하지 않아 시스템이 알 수 없는 상태로 빠지는 경우

### 예방법

- 엄격한 입력값 검증 및 위험 문자 처리
- 중앙화된 에러처리 로깅, 모니터링, 알림체계구축
- 전역 예외 핸들러 설치
- 위협 모델링 및 보안 설계 리뷰
- 코드 리뷰 정적 분석, 스트레스 침투 테스트

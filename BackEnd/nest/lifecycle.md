# Life Cycle이란?
Middleware → Guard → Pre-Interceptor → Pipe → Controller → Post-Interceptor → Response
순서로 진행됩니다
### Middleware
- 전역으로 적용된 미들웨어가 가장 먼저 실행됩니다
- 미들웨어는 요청을 처리하기 전과 후에 실행될 수 있고 주로 요청 로깅, 본문 파싱, CORS 설정 등을 담당합니다
### Guard
- 주로 인증 및 권한 부여를 처리합니다
- 사용자의 권한을 체크하는 로직이 들어가며, 요청이 컨트롤러에 도달하기 전에 실행됩니다
### InterCeptor
- 인터셉터는 컨트롤러에 도달하기 전과 응답을 클라이언트에 보내기 전에 바인딩할 수 있습니다
- 주로 로깅, 캐싱, 응답 변환할 때 사용 됩니다
- 'NestInterceptor' 인터페이스를 구현하여 작성합니다
### Pipe
- 요청이 들어오는 파라미터의 유효성 검사 및 형 변환을 담당합니다 
- 주로 데이터 검증과 변환을 담당합니다
- 'PipeTransform' 인터페이스를 구현하여 작성합니다
### Exception Filter
- 예외 처리를 담당합니다
- 예를 들어, 에러 메세지를 원하는 형태로 가공하여 응답할 수 있고 일관된 형태의 응답 메세지를 보낼 수 있습니다

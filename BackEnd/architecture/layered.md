# Layered 아키텍처란?
- 각 구성 요소들이 관심사 분리를 달성하기 위해 책임을 가진 계층으로 분리 한 아키텍처입니다
- 하나의 계층에 여러 관심사가 존재하면 응집도는 떨어지고 결합도는 높아지기 때문에 이를 보완합니다
- 재사용성과 유지보수성이 높습니다
### Layered 아키텍처의 의존성
- 한 계층에서 자신의 책임외의 행위는 하위 계층에 의존적인 구조입니다
- 하위 계층은 상위 계층에 대한 어떤 지식이나 정보가 없어야 합니다
### 예시
- Presentation Layer: 사용자의 요청이나 응답을 처리하는 책임
- Business Layer: 비즈니스 로직을 수행하는 책임
- Persistence Layer: DB와 상호작용(데이터 CRUD)하는 책임
- Database Layer: 실제 DB가 존재하는 계층

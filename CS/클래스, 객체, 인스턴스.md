# 클래스(Class), 객체(Object), 인스턴스(Instance)란?

### 클래스(Class)
- 사용자 정의 데이터 유형으로 데이터 멤버 및 멤버 함수가 포함되어 있으며 해당 클래스의 객체를 생성하여 접근하고 사용할 수 있습니다
- 객체의 추상적인 영역을 담당합니다
- 현실세계의 문제점을 해결하기 위해 도출한 개념이나 사물을 객체로 도출했을 때 객체가 가지는 공통적인 속성과 행동을 클래스에 정의하게 됩니다

```ts
class User {
  constructor(
    public name: string,
    public age: number,
  ) {}

  greet() {
    console.log(`안녕하세요, ${this.name}입니다.`);
  }
}
```

### 객체(Object)
- 클래스를 통해 생성할 대상입니다
- 생성자를 통해 구현체가 되지 않았으며 클래스의 타입으로 명시된 상황입니다. 메모리에 올라올 구현체를 위한 참조만 가집니다
- 속성(상태)과 행동(메서드)을 가진 실체를 통틀어 부르는 넓은 개념으로, 클래스로 만든 것뿐 아니라 배열이나 함수처럼 클래스 없이 만들어진 것도 객체라고 부릅니다

### 인스턴스(Instance)
- 클래스의 생성자(`new`)를 통해 실제로 구현체가 되어 메모리에 할당된 객체입니다
- "이 객체는 어떤 클래스의 인스턴스다"처럼, 특정 클래스로부터 만들어졌다는 관계를 강조할 때 인스턴스라는 표현을 씁니다

```ts
const user = new User("경진", 19); // user는 User 클래스의 인스턴스
```

### 셋의 관계
- **클래스**: 속성과 행동을 정의해 둔 설계도
- **객체**: 클래스를 통해 만들 수 있는 대상을 가리키는 넓은 개념 (아직 구현체가 되지 않은 상태도 포함)
- **인스턴스**: 그중에서도 생성자를 통해 실제로 메모리에 구현체로 만들어진 객체
- 즉 인스턴스는 객체에 포함되는 개념이며, "어떤 클래스로부터 만들어졌는지"를 강조할 때 인스턴스라는 말을 씁니다

```text
class(설계도) --new--> instance(구현체가 된 객체) ⊂ object(더 넓은 개념)
```

# SOLID란?
객체지향 프로그래밍을 통해 더 좋은 소프트웨어를 만들기 위해 SOLID 원칙이라는 설계 원칙이 등장했습니다
### SRP - 단일 책임 원칙
- 하나의 클래스는 하나의 책임만 가져야합니다
- 클래스가 변경되는 이유는 단 하나여야 합니다
### OCP - 개방-폐쇄 원칙
- 기존 코드를 변경하지 않고 확장할 수 있도록 설계해야합니다
- 새로운 기능 추가 시 기존 코드를 수정하지 않고도 구현이 가능해야 합니다
### LSP - 리스코프 치환 원칙
- 자식 클래스는 부모 클래스를 대체할 수 있어야 합니다
- 하위 클래스가 상위 클래스의 기능을 변경하면 안됩니다
### ISP - 인터페이스 분리 원칙
- 클라이언트가 사용하지 않는 인터페이스에 의존하지 않도록 해야합니다
- 하나의 큰 인터페이스보다는 여러 개의 작은 인터페이스로 분리하는 것이 좋습니다
### DIP - 의존성 역전 원칙
- 상위 모듈이 하위 모듈에 의존하는 것이 아니라, 반대로 하위 모듈이 상위 모듈에 의존해야 합니다
- 의존성을 줄이기 위해 인터페이스를 활용하는 것이 일반적입니다
